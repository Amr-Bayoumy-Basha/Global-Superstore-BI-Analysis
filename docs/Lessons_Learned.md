# Lessons Learned

## Technical Lessons

1. **Model quality determines reporting quality**
   - Clean schema and relationship design had greater impact than visual complexity.

2. **Advanced DAX requires context discipline**
   - Filter context and ranking logic are critical when building dynamic Pareto and weighted metrics.

3. **Time intelligence needs guardrails**
   - Rolling and YoY logic can mislead if partial-period behavior is not controlled.

4. **Documentation is an engineering asset**
   - Measure documentation accelerated validation and improved maintainability.

## Business Lessons

1. **Top-line growth can hide structural issues**
   - High churn and return behavior can reduce realized long-term value.

2. **Not all discount-driven sales are healthy sales**
   - Deep discounting can create revenue without sustainable margin.

3. **Segment-level actions outperform generic actions**
   - Customer-tier and segment-specific interventions are more actionable.

4. **Portfolio decisions require multi-metric views**
   - Profit-only ranking misses risk and scale dimensions.

## Portfolio Lessons

- Recruiters scan for clarity and role relevance in seconds.
- Hiring managers evaluate business impact framing quickly.
- Senior reviewers want reproducible logic and structured documentation.

Balancing these three audiences is essential for a high-performing GitHub portfolio repository.
